Different kudos-tie cut-off

Last compiled on november, 2021


Different kudos-tie classification

In our main analyses, a kudos-tie \(X_{ij}^{(h)} = 1\) if actor i gave \(\ge 1\) kudos to actor j; otherwise 0.

We incrementally increased the number of kudos required for \(X_{ij}^{(h)} = 1\). We checked how this would affect the network properties of the resulting model (e.g., density, average degree, Jaccard indices). We increased the cut-off value until the kudos network became too unstable (we aimed at an average Jaccard index > .3).

We do this for club 2, in which we found an indegree effect on frequency, and where kudos-ties were relatively stable (average Jaccard index over periods = 0.75; see below).

load("clubdata_rsiena.Rdata")
mydata <- clubdata_rsiena[[2]]  # grab club

print01Report(mydata, modelname = "files/robustness/test")
